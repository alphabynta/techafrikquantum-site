# i18n Design — TechAfrikQuantum Site
Date: 2026-03-06

## Goal
Auto-detect the user's device language and display all site content in that language. Provide a manual switcher in the nav to override. Supported languages: English (en), French (fr), Arabic (ar).

## Approach
Option A — data-i18n attributes + single JS translation file. No external dependencies, works on GitHub Pages.

## Architecture

```
js/
  i18n.js          — translations object + detection + switcher logic
style.css          — existing + [dir="rtl"] overrides for Arabic
*.html             — data-i18n="key" on every translatable element
```

## Language Detection & Switching

1. On page load, read `localStorage.lang`
2. If not set, read `navigator.language` — map to `en`, `fr`, or `ar`
3. Fallback to `en` for any other locale
4. Set `<html lang="…" dir="ltr|rtl">`
5. Query all `[data-i18n]` elements, replace `textContent` with translation value
6. Nav switcher (EN / FR / AR pills) saves choice to `localStorage` and re-applies

## Nav Switcher HTML

```html
<div class="lang-switcher" aria-label="Language">
  <button class="lang-btn" data-lang="en">EN</button>
  <button class="lang-btn" data-lang="fr">FR</button>
  <button class="lang-btn" data-lang="ar">AR</button>
</div>
```

Active lang button gets class `lang-btn--active` (amber color).

## Constraints

- Static site — no server-side rendering
- No external i18n libraries
- Arabic font: load Noto Sans Arabic from Google Fonts (already has Arabic glyphs)
- All 6 pages share the same nav template, so switcher HTML is added to each
